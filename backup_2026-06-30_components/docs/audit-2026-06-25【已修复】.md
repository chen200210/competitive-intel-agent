## 🔴 确认 Bug

### BUG #1: `raw_candidates` 为 `None` 时崩溃

**文件**: `src/agents/scorer.py:194` | **严重度**: 中

```python
raw_candidates = result.get("candidates", {})  # line 194
# ...
raw = raw_candidates.get(str(i), {})            # line 202 → AttributeError if None
```

当 AI 返回 `{"candidates": null}`（Pydantic 重试耗尽后的有效 JSON）时，`dict.get(key, default)` 返回 `None` 而非 `{}`（因为 key 存在）。第 202 行的 `.get()` 调用崩溃。此代码在 try/except 块之外，异常会传播到调用方。

**修复方向**: `raw_candidates = result.get("candidates") or {}`

### BUG #2: AI 调用失败时后备路径缺少必要字段

**文件**: `src/agents/scorer.py:192` | **严重度**: 中

```python
except Exception:
    return candidates[:top_n]  # 缺少 ai_summary / ai_score / ai_score_breakdown
```

当 `agent.run()` 抛异常时，直接返回原始候选列表的前 N 条。这些条目没有 `ai_summary`、`ai_score`、`ai_score_breakdown` 字段。briefer.py:130 将其序列化为 JSON 传给 briefer agent，prompt 期望有 `ai_summary` 字段——缺失时 AI 排版质量下降。

**修复方向**: 给后备条目补上默认字段 `ai_summary=""` / `ai_score=0` / `ai_score_breakdown={}`

## 🟡 低严重度

### BUG #3: `is_overseas()` 硬编码，与 `is_domestic()` 不一致

**文件**: `src/pipeline/source_constants.py:181` | **严重度**: 低

```python
def is_overseas(source):
    return normalize_source(source) == NewsSource.POCKET_GAMER  # 硬编码

def is_domestic(source):
    return normalize_source(source) in DOMESTIC_SOURCES  # 用集合，自动扩展
```

新增海外源（如 gamesindustry.biz）时，`is_domestic()` 自动生效，但 `is_overseas()` 仍需手动改代码。`scorer.py` 的海外保障（Pass 2 回填）依赖 `is_overseas()`，新增源会静默失效。

**修复方向**: 改为 `normalize_source(source) in OVERSEAS_SOURCES`

### 🔴 BUG #9: _select_top_n Pass 3 来源键不一致（FIX #6 不完整）

**文件**: `src/agents/scorer.py:766` | **严重度**: 高

### 🔴 BUG #10: _apply_score_fallback N=4 边界情况

**文件**: `src/agents/scorer.py:267` | **严重度**: 中

`max_above = int(4 * 0.20) = 0`，`if max_above > 0:` 跳过顶部封顶。4 条候选时如有分数 >60 无法修正，且 fallback 后无重新检查。分布违规静默通过。

**修复方向**: 去掉 `if max_above > 0:` 守卫，或改为 `if max_above >= 0:` 并处理 0 的情况（封顶全部 >60 的条目）。

### 🔴 BUG #11: 时效性检测硬编码年份 2025

**文件**: `src/agents/scorer.py:197` | **严重度**: 中

`re.search(r'去年|2025', body)` 硬编码了 2025。当前是 2026 年所以没问题，但 2027 年后所有 2026 年的旧闻将被误标为"未标注"。

**修复方向**: 改为动态年份匹配 `re.search(r'去年|' + str(current_year - 1), body)` 或直接用 `\d{4}` 匹配并比较。

### 🟡 BUG #12: AI 返回字符串 score 时类型不安全

**文件**: `src/agents/scorer.py:429` | **严重度**: 低

`raw.get("score", 0) or 0` 不拦截字符串。Pydantic v2 schema 校验可挡住大部分情况，但 `_schema_errors` 路径下仍可能传入字符串 score，后续 `>=` 比较抛 `TypeError`。

**修复方向**: `int(raw.get("score", 0) or 0)` 强制类型转换。

### 🟡 BUG #13: _fmt_body 每候选重复调用

**文件**: `src/agents/scorer.py:327-330` | **严重度**: 低

列表推导中 `_fmt_body()` 对 body 和 signals 各自调用一次，完全相同的参数。fatigue='downgraded' 时前缀拼接和截断各执行两次。

**修复方向**: 提取为局部变量复用。

### 🟡 BUG #14: score_news.py 缺少 signals 字段

**文件**: `scripts/score_news.py:222` | **严重度**: 低

`batch_score()` 构造 items_json 时未注入 signals 字段，但 summarizer.yaml prompt 明确提及此字段。AI 可能困惑或凭空编造信号值，导致评分质量低于主流水线。

**修复方向**: `batch_score()` 也调用 `_extract_body_signals()` 并注入 items_json。
